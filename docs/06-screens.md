## 4. 状態設計（各画面共通）

すべての一覧・詳細画面は4状態を実装する: **loading（Skeleton）/ empty（イラスト+CTA）/ error（再試行）/ success**。
EmptyState の文言はオンボーディング動線を兼ねる（例: ログ一覧が空→「最初の一杯を淹れてみましょう →」）。
